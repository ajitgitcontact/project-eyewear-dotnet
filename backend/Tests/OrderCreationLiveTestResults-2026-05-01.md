## Bug Found And Fixed During Live Test

Initial live success requests failed during `CustomerOrderId` generation because EF Core rejected the `INSERT ... ON CONFLICT ... RETURNING` statement through `Database.SqlQuery<int>().SingleAsync()` as non-composable SQL.

Fix:

- `CustomerOrderIdGeneratorService` now executes the PostgreSQL upsert through the underlying `DbCommand`.
- The command is attached to the current EF transaction with `CurrentTransaction.GetDbTransaction()`.
- Live tests passed after rebuilding and rerunning against the migrated database.

## Not Covered Live

These cases require controlled clock/database mutation and were not changed in the live database run:

- New server local day starts at `00001`.
- Sequence limit above `99999`.
- Forced mid-flow failures for every rollback step.

Rollback behavior was still exercised indirectly by validation and the initial generator failure, which left no successful partial order response.
